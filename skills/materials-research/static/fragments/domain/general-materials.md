# Domain: General Materials (Fallback)

Use when no specific material family matches the user's work. This is the generic fallback fragment. Before proceeding, clarify: material system, engineering use case, main performance target, available evidence, and target output format.

## Core logic

General materials papers need to connect:

1. Material system identification and design rationale.
2. Processing or fabrication route.
3. Characterization linking structure to properties.
4. Performance under service-relevant conditions.
5. Mechanism or model explaining observed behavior.

## Key evidence categories

- Composition and structure: XRD, FTIR, XPS, EDS, elemental analysis.
- Microstructure and morphology: SEM, TEM, optical microscopy, porosity, grain/particle size.
- Mechanical: hardness, tensile, flexural, impact, fatigue, creep.
- Thermal: TGA, DSC, thermal conductivity, expansion.
- Durability: corrosion, aging, weathering, chemical resistance, cyclic loading.
- Application-specific: any test directly relevant to the intended engineering use case.

## Clarification prompts

If the user has not specified, ask:

1. What is the primary material system (composition, class)?
2. What is the engineering use case or application target?
3. What is the main performance metric or property of interest?
4. What characterization or experimental data are available?
5. What is the target output (manuscript section, figure, data package, review)?

## Reviewer risks

- Vague material class: "advanced material" without specifying composition or structure.
- Missing property-performance link: properties measured but not connected to application requirements.
- Insufficient mechanism: empirical trends reported but no physical or chemical explanation.
- Incomplete characterization: key technique for the claimed mechanism absent.
- Overclaiming from limited data: broad conclusions from a narrow experimental matrix.
- No benchmark or reference comparison: performance claimed without baseline or literature context.
