# Proofreading Checklist

Final mechanical checks before returning polished text. Execute after all
substantive edits are complete.

---

## Sentence-Level Checks

| Check | Rule | Fix |
|---|---|---|
| Sentence length | ≤ 30 words target, 35 words hard ceiling | Split long sentences |
| One proposition | Each sentence has one main point | Split compound sentences |
| Subject-verb proximity | Keep verb close to its subject | Rearrange clause order |
| No comma splices | Two independent clauses need period/semicolon/conjunction | Replace comma with period |
| No fragments | Every sentence has a subject and verb | Complete the fragment |
| No dangling modifiers | Modifier must attach to the correct noun | Rewrite modifier placement |
| Parallel structure | Lists and series use the same grammatical form | Make forms consistent |

### Sentence length examples

Too long (49 words):
> The WER modified emulsified asphalt, which was prepared with 20% WER content
> and cured at 60°C for 24 h, showed a bonding strength of 0.41 MPa that was
> 73.2% higher than the base asphalt measured under the same conditions using
> the pull-off test according to ASTM D7234.

Fixed (2 sentences):
> The WER-EA with 20% WER content showed a bond strength of 0.41 MPa after
> curing at 60°C for 24 h (ASTM D7234). This was 73.2% higher than the base
> asphalt.

## Abbreviation Checks

| Check | Rule |
|---|---|
| First use | Define at first use: `waterborne epoxy resin (WER)` |
| Consistency | Use the same abbreviation throughout |
| Abstract | Re-define in abstract if it stands alone |
| Figure captions | Define abbreviation in caption if figure may be viewed independently |
| Standard abbreviations | SI units (MPa, °C, cm) do not need definition |

### Common materials abbreviations

| Full form | Abbreviation | Define? |
|---|---|---|
| Waterborne epoxy resin | WER | Yes, at first use |
| Waterborne epoxy modified emulsified asphalt | WER-EA | Yes |
| Construction and Building Materials | CBM | Yes, unless journal name |
| Cement and Concrete Composites | CCC | Yes, unless journal name |
| Fourier-transform infrared spectroscopy | FTIR | Yes |
| Scanning electron microscopy | SEM | Yes |
| X-ray diffraction | XRD | Yes |
| Thermogravimetric analysis | TGA | Yes |
| Differential scanning calorimetry | DSC | Yes |
| American Society for Testing and Materials | ASTM | Yes |
| Megapascals | MPa | No (SI unit) |

## Number and Unit Checks

| Check | Rule | Example |
|---|---|---|
| Space before unit | Always a space | `25 cm` not `25cm` |
| No space for degree | No space before ° | `60°C` not `60 °C` |
| En dash for ranges | Use – not - or to | `10–20%` not `10-20%` |
| Numerals for measurements | Always numerals | `3 specimens` not `three specimens` |
| Spell out at sentence start | Spell out numbers beginning a sentence | `Twenty specimens were tested...` |
| Decimal point | Use period, not comma | `0.78` not `0,78` |
| Thousands separator | Use comma (US) or space (UK) | `1,234` or `1 234` |
| SI units | Use standard symbols | `MPa`, `°C`, `cm`, `μm` |
| Unit consistency | Same unit throughout one comparison | Don't mix MPa and psi in one table |

### Unit formatting examples

| Wrong | Right |
|---|---|
| `25cm` | `25 cm` |
| `60 °C` | `60°C` (no space before degree) |
| `10-20%` | `10–20%` (en dash) |
| `three specimens` (mid-sentence) | `3 specimens` |
| `Twenty specimens were...` (start) | Correct |
| `0,78 MPa` | `0.78 MPa` |

---

## Figure and Table References

| Check | Rule | Example |
|---|---|---|
| Tense | Present tense | `Figure 1 shows...` not `Figure 1 showed...` |
| Capitalization | Capitalize when referring to a specific figure | `Figure 1`, `Table 3` |
| Abbreviation | `Fig.` for figures in running text (check journal) | `Fig. 1` or `Figure 1` |
| Consistency | Same style throughout | All `Figure` or all `Fig.` |
| Placement | Reference before or at the trend sentence | `As shown in Figure 1, the strength...` |

---

## Punctuation Checks

| Check | Rule |
|---|---|
| Period placement | Inside parentheses for complete sentences in parentheses; outside for partial |
| Comma before "which" | Use comma: `the material, which was tested...` |
| No comma before "that" | No comma: `the material that was tested...` |
| Oxford comma | Use consistently (most materials journals accept either) |
| Semicolon | Join related independent clauses; separate items in complex lists |
| Colon | Introduce lists, explanations, or elaborations |
| Hyphen vs. en dash | Hyphen for compounds: `water-based`; en dash for ranges: `10–20%` |

---

## Consistency Checks

| Check | Rule |
|---|---|
| Spelling variety | All American or all British (not mixed) |
| Abbreviation style | Same abbreviations throughout |
| Number style | Same format for similar items |
| Unit style | Same units for same quantities |
| Citation style | Same format throughout |
| Terminology | Same term for same concept |
| Capitalization | Consistent for technical terms |

---

## Final Checklist

### Before returning polished text

- [ ] No sentence exceeds 35 words.
- [ ] All abbreviations defined at first use.
- [ ] Units use SI with proper spacing.
- [ ] Figure/Table references in present tense.
- [ ] No double spaces after periods.
- [ ] Consistent British/American English throughout.
- [ ] All references verified against source.
- [ ] All quantitative values match the data.
- [ ] No comma splices.
- [ ] No sentence fragments.
- [ ] Parallel structure in all lists.
- [ ] En dashes for ranges (10–20%).
- [ ] No dangling modifiers.
- [ ] Subject-verb agreement.
- [ ] Consistent citation format.
- [ ] Overclaim flags documented in risk note.
- [ ] Tense correct for each section.
- [ ] Hedging calibrated to evidence level.

### Output format

Return:

1. **Polished text** — the cleaned manuscript section.
2. **Risk note** — flagged issues:

```
## Claim Risk Note

- [OVERCLAIM] Line 12: "significantly improved" → needs p-value
- [TENSE] Line 27: Methods uses present → changed to past
- [BOUNDARY] Line 35: "optimal" without durability → added qualifier
- [CITATION] Line 42: Review cited for finding → recommend original
```

3. **Change log** (optional) — for revision rounds.
