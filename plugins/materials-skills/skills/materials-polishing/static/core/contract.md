# Polishing Contract

Every polishing path must preserve:

- data, units, significant figures, error bars, and test conditions;
- specimen dimensions, processing parameters, and test standards;
- citation keys, citation intent, LaTeX, notation, and abbreviations;
- uncertainty, limitations, evidence boundaries, and author meaning.

Improve only:

- sentence clarity and paragraph flow;
- evidence-weighted verbs and hedging;
- section-appropriate tense and information order;
- journal-appropriate tone and terminology consistency.

Flag rather than silently fix:

- unsupported novelty or priority;
- missing statistical evidence;
- vague or unsupported mechanism claims;
- sustainability claims without a functional or comparison boundary;
- engineering applicability without constructability or field evidence;
- ambiguous domain terminology that cannot be resolved from the source.

Do not invent data, experiments, citations, mechanisms, statistics, standards,
journal facts, novelty, or completed actions. Use a visible risk note or
`[needs evidence: ...]` marker when the missing input changes scientific
meaning.

When a domain fragment is loaded, its terminology and evidence rules refine
this contract but never override these preservation rules.
