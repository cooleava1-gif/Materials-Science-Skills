# Section: Abstract

The abstract is a mini-paper:

`context/problem -> gap/objective -> approach -> key results -> implication`

It should answer:

1. What question was addressed?
2. How was it addressed?
3. What was found?
4. Why should anyone care?

Some journals require a strict abstract format. Follow the journal if it conflicts with the generic pattern. (See the loaded `journal/*.md` fragment for journal-specific constraints.)

## Materials-science-specific polishing priorities

- Cut sentences that summarize background that the title already implies (e.g., "Materials are important for engineering applications").
- Make the gap and the contribution one short, locatable sentence each.
- The last sentence should state significance, not repeat the result.
- Preserve all quantitative claims: compressive strength values, improvement percentages, testing conditions.
- Ensure material system is named specifically (e.g., "GO-modified cement paste" not "nanomaterial-modified cementitious composite").
- Verify that processing parameters mentioned (dosage, temperature, curing time) match the Methods section.

## Common failure modes in materials abstracts

- Vague material descriptions: "a novel material" instead of "carboxyl-functionalized graphene oxide at 0.03 wt%"
- Missing test conditions: "improved strength" without specifying testing age, specimen type, or standard
- Overclaiming mechanism: "GO promotes nucleation" without direct evidence (should be hedged: "may promote" or "is consistent with")
- Mixing Results and Discussion: interpreting XRD peaks in the abstract instead of just reporting phase changes
