# Section: Results

Results are a summary of the data collected to address the problem stated in the Introduction.

Results writing should:

- stay mainly in past tense
- report what was observed, under what conditions, and with what quantitative support
- use statistics correctly and sparingly
- use supplementary data sparingly
- present data in logical order (not necessarily chronological order of testing)

Results should answer `what happened`, not `what it ultimately means`.

## Sentence syntax (Results vs Discussion)

Results sentences usually report:

- `was detected`
- `increased` / `decreased`
- `showed` / `exhibited`
- `enabled` / `achieved`
- `was observed`
- `reached` (e.g., "compressive strength reached 65 MPa")

Do not let a Results paragraph drift into Discussion syntax (`may reflect`, `suggests that`, `is likely due to`) unless the transition is intentional and clearly marked.

## Materials-science-specific requirements

### Mechanical properties
- Report mean ± standard deviation (e.g., "65.3 ± 2.1 MPa").
- Specify testing age (3 d, 7 d, 28 d).
- Compare to control mixture explicitly (e.g., "25% higher than the control mixture M0").
- If reporting multiple properties (compressive, flexural, split tensile), organize by property or by mixture, not randomly.

### Microstructure characterization
- SEM: describe observed features (morphology, distribution, agglomeration), not just "SEM was performed".
  - Good: "GO sheets were uniformly dispersed in the cement matrix with minimal agglomeration (Fig. 3a)."
  - Bad: "The microstructure was observed by SEM."
- XRD: report phase identification and relative quantities if Rietveld refinement was used.
  - Good: "XRD analysis confirmed the presence of C-S-H, CH, and unhydrated C₃S. The intensity of CH peaks decreased with increasing GO dosage, indicating accelerated hydration."
  - Bad: "XRD patterns were obtained."
- TGA/DSC: report mass loss percentages and temperature ranges for specific decomposition events.
  - Good: "TGA showed a 15% mass loss between 100-200 °C (evaporable water) and 8% mass loss between 400-500 °C (CH decomposition)."
  - Bad: "Thermal analysis was performed."

### Durability properties
- Report test conditions (exposure time, solution concentration, temperature).
- Compare to control and specify improvement percentage.
- If reporting multiple durability indicators (permeability, carbonation, chloride ingress), organize logically.

## Common failure modes in materials Results

- Interpreting findings inline instead of in Discussion (e.g., "The strength increase is due to GO nucleation effect" → move to Discussion).
- Citing supplementary figures when the result should stand in the main text (e.g., "see Supplementary Fig. S3" for a key compressive strength result).
- Vague comparisons without effect size: "higher than control" → specify "25% higher than control (65 MPa vs. 52 MPa)".
- Reporting data without context: "compressive strength was 65 MPa" → compare to control, specify testing age, mention if this meets target performance.
- Mixing Results and Discussion in the same paragraph.
- Presenting characterization data without linking to the research question (e.g., showing XRD patterns without explaining what they reveal about the gap stated in Introduction).

## Polishing priorities

- Ensure all quantitative claims include units and error bars where applicable.
- Verify that Results address the research question stated in Introduction.
- Check that all figures and tables are cited in text and appear in logical order.
- Remove any interpretation that belongs in Discussion.
- Ensure terminology consistency: if "GO" was defined in Introduction, do not switch to "graphene oxide" in Results.
