# Paragraph Flow

Use this reference when the user asks whether a paragraph flows, makes sense, or is clear.

## Core principle

One paragraph carries one message, and the first sentence announces it. Flow is not decoration: a paragraph flows when an external reader can identify the message, see how the first sentence forecasts it, and see how each following sentence serves it.

Allowed paragraph messages:

- context
- gap
- approach
- result
- comparison
- mechanism
- implication
- limitation

## Reader test

Read as a skeptical but fair external reader and ask:

1. After reading the paragraph, can you state its message in one sentence?
2. Does the first sentence forecast what the paragraph is doing?
3. Does each sentence serve the first sentence, without drifting?
4. Does the transition between paragraphs advance the argument, not just continue it?
5. Is any sentence carrying material that belongs in another paragraph?

## reverse outlining

For a section-level flow check:

1. After each paragraph, write a one-sentence summary of its message.
2. Read the summary sequence and check whether it forms a coherent argument chain.
3. Locate any break in the chain where the logic jumps.
4. Locate any paragraph whose summary contains two messages.
5. Split multi-message paragraphs, merge fragments, or reorder to restore the chain.
6. Rewrite the transitions so each paragraph opening pushes the argument forward.

If reverse outlining is hard, the section probably has a hidden structure problem.

## reverse-outline example: splitting a two-message paragraph

Original paragraph (two messages: result + mechanism):

> The bonding strength of WER-EA tack coat increased from 0.43 MPa to 0.55 MPa at 10% epoxy content. This improvement is due to the oxirane groups reacting with the asphalt interface, forming a more continuous interfacial morphology that improves load transfer.

Split and repaired:

> **Result paragraph**: The bonding strength of WER-EA tack coat increased from 0.43 MPa to 0.55 MPa at 10% epoxy content, measured under dry conditions at 25 °C.
>
> **Mechanism paragraph**: This strength increase is consistent with a more continuous interfacial morphology: FTIR shows the oxirane peak at 915 cm⁻¹ decreases with curing time, and SEM images show fewer voids at the interface at 10% epoxy than in the control.

Transition added: "This strength increase is consistent with ..." links the result paragraph to the mechanism paragraph and advances the argument.

## Materials-specific paragraph patterns

Results paragraph, three-sentence mode: `data -> comparison -> mechanism hint`. Report the number, place it against a baseline or prior value, then signal what the data implies without overclaiming the mechanism.

Discussion paragraph, three-sentence mode: `finding -> literature positioning -> boundary`. State the finding, position it against prior work, then state what remains unproven or condition-specific.

## Repair moves

- Message unclear: write the one-sentence summary first, then revise the paragraph to deliver it.
- Broken chain: add an explicit transition sentence, or reorder paragraphs to restore the logic.
- Multi-message paragraph: split it so each new paragraph carries one message only.
- Vague transition: replace with an explicit relation such as `therefore`, `however`, `by contrast`, `for example`, or `as a result`.
- Keep paragraph openings claim-first unless the section needs a brief setup.
