# Conclusion Examples

`One skeleton plus overclaim replacements. Distilled from conclusion-boundary.md Move 1-4 and published-article-patterns Pattern M/N. Conclusions summarize verified contributions — no new data, no new citations.`

## Skeleton: contribution → evidence → implication → boundary → future work

```latex
% Move 1: contribution — specific property-mechanism-service link
This study demonstrates that [材料变量] can improve [性能] within [配比/工艺窗口] for [应用], reaching [数值] under [测试条件].

% Move 2: decisive evidence — the characterization that supports the claim
[表征技术] evidence supports a mechanism in which [有界解释], which is consistent with the measured [性能变化].

% Move 3: implication — what the result means for design, bounded
Within the tested [配比/工艺/老化] range, [材料] shows potential for [有界应用], because [证据基础] was limited to [测试标准/样品尺度].

% Move 4: limitation — name the missing condition before the reviewer does
The conclusion is limited by [样品尺度/测试时长/标准/缺乏现场验证]; because [表征/测试] was not included, [具体机制] should be regarded as a supported interpretation rather than a confirmed cause.

% Move 5: future work — the next experiment that would strengthen the claim
Future work should verify [耐久性/服役条件] using [标准/协议] and [现场/更大尺度样品], and add [表征方法] to distinguish [机制A] from [替代机制].
```

## Overclaim replacements

Use these before submitting — reviewers scan conclusions for these words.

- Replace `proves the mechanism` with `supports a mechanism in which [evidence-bounded explanation]`.
- Replace `solves the durability problem` with `improves [durability metric] under [tested protocol]`.
- Replace `can be widely applied` with `is promising for [application] after [missing validation] is addressed`.

## Usage

- Move 1 must specify the material system, the property, AND the test condition — a contribution without a condition is an overclaim.
- Move 4 must state the limitation explicitly, not hide it inside "future work".
- Avoid `excellent`, `significant`, `novel`, `field-ready` unless the data and comparison justify them.
