# Introduction Variant 1 — Engineering Bottleneck

`Use when the gap is a performance, mechanism, or durability limit in an applied material. Distilled from introduction-gap-ladder.md Move 1-4 and published-article-patterns Pattern D.`

```latex
% Move 1: field progress — what the field can already do
Recent studies on [材料家族] have improved [性能/应用] through [主导策略] ([文献范围]).

% Move 2: current performance boundary — name the ceiling with a number
For [应用场景], however, [材料] still achieves only [性能数值] under [服役条件], which is below the [工程需求] threshold of [需求数值].

% Move 3: evolution of modification strategies — show the trajectory, not a list
Modification strategies have evolved from [早期策略] to [近期策略], each raising [性能指标] but leaving [耐久性/机制/界面问题] unresolved ([文献范围]).

% Move 4: unresolved gap — name the missing evidence type, not "few studies exist"
The current evidence is limited by [缺失的表征/测试标准/样品条件], making it difficult to separate [机制A] from [替代解释] for [材料体系] under [服役条件].

% Move 5: this study's design + validation matrix
Therefore, this study investigates [材料变量] using [测试矩阵] and [表征方法] to clarify [有界机制/性能关系] within [配比/工艺/服役边界].
```

## Usage

- Move 2 must give a number — a bottleneck without a quantitative ceiling is a slogan.
- Move 3 is a trajectory, not a paper list. Synthesize what each strategy left behind.
- Move 4 must name the missing evidence type (characterization, standard, condition), not just "few studies".

## Gap quality check

The ladder is weak if Move 5 could apply to any material. Anchor the entry to:
- a specific property ([性能指标]),
- a specific mechanism evidence ([表征方法]),
- a specific service condition ([服役条件]).
