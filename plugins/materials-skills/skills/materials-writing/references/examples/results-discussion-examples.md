# Results & Discussion Examples

`Two skeletons for results-discussion paragraphs. Distilled from results-discussion-evidence-chain.md Move 1-4 and published-article-patterns Pattern H/I/J/K. Do not discuss mechanism before the result is clear.`

## Skeleton A: observation-driven

`Use when a single observation leads to a mechanism interpretation.`

```latex
% Move 1: observation — report what changed, cite the figure, no interpretation yet
As shown in Fig. [x], [性能指标] changed from [数值A] to [数值B] when [变量] increased from [水平A] to [水平B].

% Move 2: quantitative result — effect size, uncertainty, comparison
The maximum [性能] reached [数值] ± [不确定度] at [条件], corresponding to a [百分比] change relative to the control ([对照数值]).

% Move 3: mechanism evidence — name the characterization support before interpreting
[表征技术] showed [可观察特征] for the [配比] group (Fig. [x]), consistent with [机制], which helps explain the [性能变化].

% Move 4: alternative explanation exclusion — show what else could explain it
A possible alternative is [替代解释], but [对照/测试/表征] reduces this possibility because [证据]; because [未测量变量] was not measured, the data support an association rather than a confirmed cause.
```

## Skeleton B: comparison-driven

`Use when the key move is comparing your result to a literature baseline.`

```latex
% Move 1: comparison with literature baseline — same material system, condition, scale
The [性能数值] obtained here is [百分比] higher than the [基线数值] reported by [文献引用] for [材料体系] under [条件].

% Move 2: difference — name what is different and by how much
The difference is most pronounced at [条件/配比], where our system [特征] while the literature baseline [特征].

% Move 3: mechanism explanation — why the difference exists, supported by characterization
[表征技术] indicates that [结构差异] is associated with [机制], which is consistent with the [性能差异]; this agrees with [文献] though our system showed [差异点].

% Move 4: boundary — where the comparison stops being fair
The comparison is limited to [测试条件/尺度/配比范围]; [未对齐的变量] means the difference should not be generalized beyond [边界].
```

## Usage

- Skeleton A Move 3 must use `consistent with` or `suggests`, never `confirms` for a single-technique observation.
- Skeleton B Move 1 must compare like with like (same material system, condition, scale) — an unfair comparison is a red flag.
- Both skeletons require Move 4: a results paragraph without a boundary or alternative-exclusion is incomplete.
