# Section: Results (writing)

> **Domain context**: The `domain` axis has loaded domain-specific writing guidance for [detected domain]. The reporting requirements below vary by material family — the domain guide contains characterization-specific details. Use both together.

Results report what was observed, under what conditions, with what quantitative support. Do not interpret here — that belongs in Discussion.

## Default structure

`observation -> quantitative result -> figure/table reference -> statistical report -> transition to next result`

Each subsection has a claim-first opening, then data support.

## Drafting rules

- Stay mainly in past tense.
- Report what was observed, under what conditions, with what quantitative support.
- **Do not infer mechanism from performance alone.** State the trend; leave the mechanism to Discussion.
- **Do not mix observation and interpretation in the same paragraph.** If interpretation appears, move it to Discussion or mark it for relocation.
- Link each performance trend to the figure or table that supports it; do not describe a result the reader cannot find.

## Statistical reporting

- Report `mean ± SD` (or `SE`), replicate count, and the test used: `42.3 ± 1.1 MPa (n = 5, Student's t-test, p < 0.05)`.
- `Significant` only with a stated test and p-value; otherwise use `increased`, `higher`, `improved`.
- Report effect size for comparisons, not just direction: `15% higher than the control`, not `higher than control`.
- Keep consistent significant figures across comparisons.

## Results syntax (vs Discussion)

Results sentences usually report:

- `was detected` / `increased` / `showed` / `achieved` / `reached`

Do not drift into Discussion syntax (`may reflect`, `suggests`, `is likely due to`) unless the transition is intentional and clearly marked.

## Domain characterization evidence

Pair each performance result with the corresponding characterization evidence (loaded by the `domain` axis):

- **Civil materials**: FTIR, SEM, fluorescence microscopy, rheology, TGA/DSC.
- **Ceramics**: XRD, SEM/TEM, EDS, density/porosity, Weibull analysis.
- **Metals**: OM, SEM, EBSD, TEM, XRD, fractography, hardness.
- **Polymers**: DSC, TGA, DMA, FTIR, SEM, rheology.
- **Functional**: XRD, Raman, XPS, TEM, UV-vis, electrochemical impedance.
- **Nanomaterials**: TEM (with size statistics), SEM, XRD (Scherrer), DLS, UV-vis, Raman.

## Common failure modes

- Mixing observation and interpretation in the same paragraph.
- Vague comparisons (`higher than control`) without effect size, sample size, or test.
- Per-paragraph claims without per-paragraph evidence.
- Citing supplementary data when the result should be in the main text.

## Domain checklist

- [ ] Does each key result have a corresponding characterization evidence?
- [ ] Are error bars or confidence intervals reported for all quantitative data?
- [ ] Is the comparator named for every improvement claim?
- [ ] Is interpretation kept out of Results (moved to Discussion)?
