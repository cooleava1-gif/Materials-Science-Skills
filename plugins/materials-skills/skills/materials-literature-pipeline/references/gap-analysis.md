# Gap Analysis

Use this four-step method when the user asks whether a materials topic has been
explored, whether a research gap is real, or how to support an introduction gap
statement.

## Step 1: Multi-Source Search

Search at least three query layers:

1. Exact topic phrase or exact material system.
2. Component-level or subsystem keywords.
3. Adjacent method, phenomenon, or application terms.

Record the query, source, date, and hit count. Absence of hits is only meaningful
when the search terms are recorded.

## Step 2: Decompose And Classify

If direct hits are sparse, decompose the system into sub-systems:

| Layer | Example status |
|---|---|
| Exact material system + target phenomenon | direct / absent |
| Same material family + same method | edge-related |
| Same phenomenon + adjacent material family | transferable context |
| Same method only | background |

Classify each paper as:

- `direct`: same material system and research question.
- `edge`: partial material or method overlap with extractable insight.
- `background`: context only; not evidence for novelty.
- `exclude`: unrelated to the claimed gap.

## Step 3: Extract Edge Papers

For every `direct` or high-value `edge` paper, extract:

- Citation and DOI/link.
- Material composition or system boundary.
- Method and test conditions.
- Main finding relevant to the gap.
- What can be borrowed for the user's work.
- What the paper does not cover.

## Step 4: Output Gap Report

Write a gap report with these sections:

1. Core conclusion: confirmed gap, partial overlap, or no gap.
2. Search log: exact queries, sources, hit counts, and date.
3. Subsystem landscape: table of direct, edge, background, and excluded papers.
4. Edge-paper extraction: methods, findings, and transferable limits.
5. Gap mechanism: why the gap exists and why it matters.
6. Introduction-ready gap statement: conservative English paragraph.
7. Next actions: reading, citation audit, DOE/data/figure implications.

## Guardrails

- Do not call a gap real if search coverage is narrow or undocumented.
- Do not use metadata-only records as proof that a gap exists.
- If the gap is only a time-scale, environment, dosage, or test-method gap,
  state that narrower boundary explicitly.
