# Task: Submission Package

Goal: prepare journal-facing materials after the manuscript argument is stable.

Hand off final assembly to `materials-submission`. Its required handoff is a
`submission-package/` directory containing `submission-package.yaml`,
`MANIFEST.md`, source-status stubs, and the journal-routed generated artifacts.

Common deliverables:

- Title alternatives.
- Highlights.
- Abstract.
- Keywords.
- Cover letter.
- Graphical abstract plan.
- Declaration of competing interests.
- Funding statement.
- Data availability statement.
- Reviewer suggestions if requested by the journal.
- Response to editor checks or technical screening.

Rules:

- Verify the current guide for authors before exact formatting.
- Do not invent author declarations, funding, data availability, or suggested reviewers.
- Keep highlights concrete and result-based.
- Cover letters should emphasize journal fit and evidence-supported contribution, not flattery.

If the manuscript evidence chain is weak, pause and run `reviewer-audit` before polishing the package.
