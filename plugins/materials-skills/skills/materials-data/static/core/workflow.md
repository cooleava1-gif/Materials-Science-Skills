# Workflow

1. Identify the data task: template, package, FAIR audit, or data availability statement.
2. Identify the material domain and target journal.
3. If the user provides an `experiment-record.yaml`, validate it against `_shared/core/experiment-record-schema.yaml` and use it to scaffold the dataset package.
4. Create or inspect the dataset package structure.
5. Check FAIR: findable, accessible, interoperable, reusable.
6. Flag missing metadata before manuscript claims or submission statements are drafted.
