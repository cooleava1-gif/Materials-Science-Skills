# Submission Package Protocol

1. Read the manifest and selected journal template.
2. Validate the journal, article type, required live verification, title,
   corresponding author, and FAIR-data status.
3. Read optional manuscript, figure, FAIR-data, and reviewer-routing sources.
4. Write the package manifest, cover letter, keywords, declarations, checklist,
   source stubs, and reviewer-risk record.
5. Write highlights only when the selected template requires them.
6. Render journal-specific artifacts in the checklist, grouped as
   initial-submission or revision requirements.
7. Preserve every live-verification placeholder until the user confirms the
   current publisher requirements.
