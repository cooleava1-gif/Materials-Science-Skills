# Paper type: methods / protocol / tool / algorithm

Covers methods, protocols, characterization techniques, computational workflows, and tools that enable materials research.

## Presentation arc — problem-to-solution

Best when the paper proposes a procedure or tool and must show it works and is better. Order the story as:

1. the current bottleneck or limitation,
2. the proposed method,
3. the workflow or architecture,
4. the evaluation design,
5. performance compared with baselines or standards,
6. ablation, robustness, or failure cases,
7. reuse scenarios and limitations.

## Default slide structure (adapt, do not force)

1. 标题页
2. 研究背景：当前方法的瓶颈
3. 核心问题：要解决什么
4. 方法总览 / 流程图（通常 full-width 流程图）
5. 关键设计 / 模块 / 参数
6. 验证设置：材料体系、基准方法、评价指标
7-8. 主要性能结果（与基准对比）
9. 稳健性 / 适用边界 / 失败案例
10. 方法优势与适用场景
11. 复用情况与开源/标准情况
12. 局限性
13. 总结

Use editable PPT-native tables/charts for benchmark numbers when values are explicit. Keep material names, test standards, instrument names, and metric names identical to the paper via the Terminology Ledger. The workflow/architecture slide is usually a full-width process diagram, not a two-column split.
