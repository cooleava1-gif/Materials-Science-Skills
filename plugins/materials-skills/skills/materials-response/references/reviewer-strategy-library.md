# Reviewer Strategy Library

> **Domain context**: The `domain` axis has loaded domain-specific response guidance. The strategies below are universal; the domain guide lists domain-specific reviewer concerns and response strategies.

Use this library when the reviewer comment requires a strategic response beyond simple revision. Each strategy includes the reasoning, response structure, and risk boundaries.

## 1. Novelty Challenges

### 1.1 "Incremental contribution" / "Not novel enough"

**Strategy**: Reframe contribution as bounded evidence combination, not universal novelty.

**Response structure**:
1. Acknowledge the reviewer's perspective on novelty expectations.
2. Clarify what is new: material system, evidence combination, service condition, or analytical framework.
3. Position contribution within the specific gap identified in the Introduction.
4. Avoid claiming "first" or "novel" — use "combined evaluation," "systematic assessment," or "evidence-based framework."

**Risk boundary**: If the contribution is genuinely incremental, reduce claims and add limitations rather than overstate novelty.

**Example**:
> We appreciate the reviewer's comment on novelty. The revised manuscript now positions the contribution as the combined evaluation of processing stability, microstructural evidence, performance response, and durability retention for the studied material system. We have revised the Introduction to clarify how this evidence chain differs from previous performance-only reports.

### 1.2 "Literature comparison insufficient"

**Strategy**: Add structured comparison table; if conditions differ, compare trends not absolute values.

**Response structure**:
1. Acknowledge the need for broader comparison.
2. Add comparison table with key variables (material, method, condition, result).
3. If direct comparison is impossible, explain why and compare trends.
4. Cite the comparison in Discussion to contextualize results.

**Risk boundary**: Do not force incomparable data into a ranking table.

### 1.3 "Research gap not clearly identified"

**Strategy**: Strengthen the gap-ladder in Introduction with specific evidence gaps.

**Response structure**:
1. Revise Introduction to explicitly state what is missing in the literature.
2. Link the gap to your specific contribution.
3. Add 2-3 recent references that highlight the gap.
4. Ensure the gap is material-system specific, not a generic complaint.

## 2. Method and Evidence Challenges

### 2.1 "Method selection not justified"

**Strategy**: Explain method-research question fit; acknowledge what alternative methods would add.

**Response structure**:
1. State why the chosen method addresses the research question.
2. Acknowledge what the alternative method would provide.
3. Add limitation or future work if the alternative is important.
4. Cite standards or precedents for the chosen method.

**Example**:
> The pull-off test was selected because the study focuses on tensile interlayer adhesion under tack-coat application conditions. We agree that shear testing provides complementary information; this limitation is now stated in Section [x], and the conclusion is limited to tensile bond performance.

### 2.2 "Sample size too small"

**Strategy**: Report replicates, add uncertainty, reduce claim strength if needed.

**Response structure**:
1. State replicate count and statistical measures (SD, SE, CI).
2. If n is small, acknowledge limitation explicitly.
3. Reduce claim strength from "significant" to "trend" or "preliminary."
4. If possible, add more replicates; if not, explain why.

**Risk boundary**: Never claim statistical significance without adequate sample size.

### 2.3 "Statistics insufficient"

**Strategy**: Add appropriate statistical tests; if data don't support them, state limitation.

**Response structure**:
1. Add the requested statistical analysis (ANOVA, t-test, regression).
2. Report p-values, confidence intervals, or effect sizes as appropriate.
3. If the data structure doesn't support the requested test, explain why and offer an alternative.
4. Update figures with error bars and statistical annotations.

### 2.4 "Mechanism not supported by evidence"

**Strategy**: Add direct evidence (FTIR, SEM, XRD, etc.) or soften mechanism claims.

**Response structure**:
1. If evidence is available: add it to Results and revise Discussion.
2. If evidence is unavailable: soften mechanism language from "demonstrates" to "suggests" or "is consistent with."
3. Add a limitation paragraph acknowledging the mechanism is inferred.
4. Use the evidence hierarchy: direct observation > indirect evidence > inference > speculation.

## 3. Writing and Presentation Challenges

### 3.1 "English needs major revision"

**Strategy**: Acknowledge readability concern; state manuscript was fully edited; mention technical terminology.

**Response structure**:
1. Thank the reviewer for the language feedback.
2. State that the manuscript has been thoroughly revised for grammar, clarity, and consistency.
3. Mention specific areas revised (e.g., technical terminology, figure captions, methods section).
4. Do not claim professional editing unless it actually happened.

### 3.2 "Figures need improvement"

**Strategy**: Specify what was improved (resolution, labels, color scheme, error bars).

**Response structure**:
1. List specific figure improvements made.
2. If the reviewer requested a new figure, describe it and cite its location.
3. If the reviewer's suggestion was impractical, explain why and offer an alternative.
4. Ensure figure captions are updated to match revisions.

### 3.3 "Structure/organization unclear"

**Strategy**: Revise section structure; add transitions; clarify logical flow.

**Response structure**:
1. Acknowledge the organization concern.
2. Describe structural changes made (new subsections, moved paragraphs, added transitions).
3. If the reviewer suggested a specific reorganization, explain why you followed or adapted it.
4. Cite the revised structure in the response.

## 4. Scope and Framing Challenges

### 4.1 "Beyond scope" / "Should be in a separate paper"

**Strategy**: Acknowledge value; explain scope boundary; reduce claims; add future work.

**Response structure**:
1. Thank the reviewer for recognizing the value of the suggested work.
2. Explain the scope boundary of the current study.
3. Reduce claims to match the limited scope.
4. Add the suggested work as future work or a limitation.

**Example**:
> We acknowledge that additional [test] would strengthen the mechanism evidence. Because the present study was designed around [available tests/scope], this experiment is beyond the current scope. We have therefore revised the mechanism discussion to be more cautious and added [test] as future work.

### 4.2 "More suitable for another journal"

**Strategy**: Respond through scope, not prestige; strengthen journal-fit language.

**Response structure**:
1. Acknowledge the reviewer's perspective on journal fit.
2. Revise Introduction and Conclusion to emphasize journal-relevant aspects.
3. If the editor's decision is firm, do not argue excessively.
4. If resubmitting to the suggested journal, adapt the manuscript accordingly.

### 4.3 "Overclaiming" / "Conclusions too strong"

**Strategy**: Narrow claims to match evidence; add hedging language.

**Response structure**:
1. Identify the overclaimed statements.
2. Revise to match the evidence level (see claim-strength-ladder).
3. Add appropriate hedging: "suggests," "indicates," "is consistent with."
4. Ensure conclusions are bounded by the data presented.

## 5. Conflicting Reviewer Challenges

### 5.1 Contradictory requests

**Strategy**: Identify shared concern; avoid saying one reviewer is wrong; make balanced revision.

**Response structure**:
1. Identify the underlying concern behind both comments.
2. Address the shared concern with evidence.
3. If the requests truly conflict, explain the trade-off and justify your choice.
4. Thank both reviewers for their input.

**Example**:
> Reviewer 1 requested a more detailed mechanism discussion, whereas Reviewer 2 cautioned against speculative interpretation. We addressed both concerns by adding the available FTIR/microscopy evidence while revising the mechanism wording to distinguish direct observations from interpretation.

### 5.2 One reviewer enthusiastic, one critical

**Strategy**: Address the critical reviewer's concerns while preserving the enthusiastic reviewer's positive assessment.

**Response structure**:
1. Focus on the critical reviewer's specific concerns.
2. Make concrete revisions to address each concern.
3. Do not dismiss the critical reviewer by citing the positive one.
4. Use the positive assessment to strengthen the revised manuscript's framing.

## 6. Data and Reproducibility Challenges

### 6.1 "Provide raw data"

**Strategy**: Provide raw/processed data package when possible; include metadata.

**Response structure**:
1. Prepare supplementary dataset with raw measurements, processed averages, replicate counts, units, and test conditions.
2. Update data availability statement.
3. If constrained, provide available subset and explain limits.
4. Ensure data format is usable (CSV, Excel with clear headers).

### 6.2 "Reproducibility concerns"

**Strategy**: Add detailed methods, parameters, and conditions.

**Response structure**:
1. Add specific equipment models, software versions, and parameter settings.
2. Describe sample preparation in step-by-step detail.
3. Include photographs or diagrams of experimental setup if helpful.
4. Cite relevant standards (ASTM, ISO, GB/T) for each test method.

## 7. Ethics and Integrity Challenges

### 7.1 "Potential plagiarism" or "Insufficient attribution"

**Strategy**: Carefully review citations; add missing attributions; clarify original contributions.

**Response structure**:
1. Review all cited and uncited references to similar work.
2. Add missing citations with clear attribution.
3. Explicitly state what is original in your work versus what is established.
4. If the concern is about self-plagiarism, explain what is new in this submission.

### 7.2 "Conflict of interest" or "Ethical concerns"

**Strategy**: Address directly and transparently.

**Response structure**:
1. Acknowledge the concern seriously.
2. Provide clear, factual response (e.g., funding disclosure, ethical approval).
3. If there is a genuine issue, correct it and explain the correction.
4. Do not be defensive; transparency builds trust.
